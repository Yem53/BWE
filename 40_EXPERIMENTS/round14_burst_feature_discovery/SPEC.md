# Round 14 — 妖币大波动爆发:特征发现 + 多角度规律挖掘 Spec

> **状态:设计待批准。Spec 阶段——批准前不写实现代码。** 设计经 brainstorming 收敛。
> 真钱研究($467,3倍杠杆):任何要上实盘的产物,需 holdout 通过 + 人工复核 + 用户确认。研究只读、不碰实盘。

## 0. 目标

在过去 ~2 个月全市场 U 本位永续的大波动爆发上,**发现"在什么时候、哪个方向、精准开仓"的可识别特征规律**,并诚实判定哪些能转化成可交易信号(入场/方向/风控),哪些是噪声。

分三个预测层(同一批事件、同一套特征,只换标签):
- **① 方向(两面)**:进场后 N 小时 创新高(做多收益)**和** 狂跌(做空收益)都测 → 决定做多还是做空。
- **② 时机**:这一刻是"刚启动"还是"接近见顶" → 进场精准度。
- **③ 风险**:前向最大逆向 / 是否失控拉飞到打爆 3 倍止损 → 规避/降仓。

## 2. 事件锚点 + 数据切分

- 锚点 = 全归档重跑后的去重事件(~118天,预计 ~3000 事件)。每个事件取"进场时刻"(默认 = 事件检测起点;时机层会另取多个候选进场点,见 §4②)。
- **封存 HOLDOUT = 最近 35 天**(05-25 往前 35 天 ≈ 04-20→05-25,约占全程 30%)。筛选/组合/调参阶段**绝不读**,只在每层最终验一次。
- **DEV = 01-27→04-20(~83 天)**,walk-forward 扩展折(train→validate→滚动,3 折)。
- (若用户选不重跑、沿用 2 月事件:holdout 缩到 ~18 天换 dev ~40 天,但单薄、结论更弱——不推荐。)
- **持久性劈半检验(地基,先做)**:把 2 个月劈两半,看"惯犯币榜"重合度。若不稳→猎场池/历史频次特征作废(纯事后幸存者偏差)。

## 3. 特征面板(全部 🟢 因果 / as-of,在"进场时刻"重算,绝不偷看未来)

**A. 形态/动力学 —— ×6 颗粒度 {1m,3m,5m,15m,30m,1h} 各算一遍:**
多窗涨幅(ATR归一)· 加速度(Δ动量,抛物线探测)· 连阳/连阴根数 · 实体/上下影比(强推力vs被拒)· 振幅扩张(本根/近N均)· 回撤深度 · 爆发前压缩(蓄势)· 距基准已涨多少(第几条腿)。
→ 多角度:同一形态在不同粒度上的值 + 跨粒度一致性(快慢周期是否共振)。

**B. 量/参与**(K线):量z分 · 爆发中量放大vs衰减(背离=力竭)· 美元成交额 · 单笔均额(鲸鱼vs散户)· taker买卖失衡。
**C. 衍生品**(特征库,按覆盖期裁切):OI水平+1h变化(新仓vs回补)· 资金费率 · premium · 大户/散户多空比。
**D. 截面/市场背景**(全归档):此刻全市场涨幅/波动榜排名(独涨vs普涨)· 同时爆发币数(狂热日)· BTC当时regime/24h。
**E. 币静态**:波动级 atr_pct(已证关键)· 流动性档 · 上市时长(新币?)· 历史爆发频次(只用截止当时)。
**F. 时间**:UTC小时(已证有用)· 距上次爆发时长。

## 4. 三层标签(向前测,特征不偷看未来)

对每个进场锚点,从 1m 路径在 H ∈ {1h,2h,4h}(可扩展)上算:`fwd_ret`、`mfe_up`(对多最好)、`mae_dn`(对空最好/对多最坏)、`bars_to_peak`。

- **① 方向(两面)**:
  - 做多分支:fwd_ret / mfe_up 为正且显著(继续涨)。
  - 做空分支:−fwd_ret / |mae_dn| 为正且显著(**进场后狂跌**)。
  - 标签 = 在该事件上"哪一边赚、赚多少",按特征分层看可预测性。
- **② 时机**:remaining-MFE 比例(前向还剩多少有利空间 / 已走完多少)+ 距前向峰值 bar 数 → "刚启动"(剩余多)vs"接近见顶"(剩余少)。时机层另取多个候选进场点(onset、+1/+2 bar、回踩点)对比。
- **③ 风险(按side)**:做空风险 = 前向最大上冲(逼空);做多风险 = 前向最大下砸;runaway flag = 逆向幅度达到 3 倍杠杆爆仓线。

## 5. 方法:路线 C 分阶段(描述 → 组合 → 受限ML → holdout)

1. **Phase 0**:重跑爆发扫描到全归档(01-27→05-25)+ 持久性劈半检验 + 读 Codex 报告 + 流动性/可交易性过滤(剔除做不了的币,见 §6)+ 在事件锚点上 as-of 构建全特征面板(含 6 粒度),sanity(无前视审计、覆盖率、基础率)。
2. **Phase 1 逐类描述筛(A→B→C→D→E→F,每层 ×3 标签)**:每个(特征,粒度)在 DEV 上看条件分位价差 + Spearman + 事件研究平均路径,**单调且跨折/跨币组一致**才标"有信号"。不调参,只判有无,出**逐类信号榜**。
3. **Phase 2 组合探索**:把各类幸存的 alpha 点拼起来(先简单 AND/打分,再受限浅树),测交互是否 1+1>2;必须嵌套CV + 打赢最好单特征,否则因复杂度否决。
4. **Phase 3 holdout 确认(每层一次)**:最终候选在封存 35 天验一次。扛不住即否决。

## 6. 流动性/可交易性过滤(猎场池排除线)

剔除"完全做不了"的:日均成交额低于阈值(实盘 >0.8% 滑点弃单,11/23 信号死于此)、MARKET_LOT_SIZE 限制下 $100 名义打不进、已下架/归零币。阈值:Phase 0 先看成交额分布再定(默认候选:日均 quote_volume < ~$300-500万 剔除),用户确认。

## 7. 反过拟合纪律(默认开,违则作废)—— 本轮尤其吃重

搜索面 = 6粒度 × ~25特征 × 多角度 × 组合 × 3层 × 多空两面 = **巨大多重比较面**,假阳性几乎必然。防线:
- holdout 全程封存,每层只验一次;DEV walk-forward,要求跨折 **+** 跨币组(妖币/中/大)一致,非单点显著。
- **持久性劈半**是猎场池/历史频次特征的前置闸。
- **BH-FDR** 修正,报"共试多少 (特征×粒度×层) 组合"。
- 因果无前视(特征只用 ≤进场bar;历史频次只用截止当时);结果好得离谱(胜率>65%或单笔>2%)先停查泄漏。
- **幸存者偏差核查**:用尽量含已下架/归零币的池子复核(尤其高ATR过滤)。
- 成本:taker来回~0.14% + 急拉成交性(>0.8%滑点弃单)。
- **诚实报负是合格主交付**;"多数是噪声"可接受且很可能。

## 9. 公开风险/诚实前提

- 这是描述性扫描转信号挖掘,强先验是"妖币爆发后方向≈抛硬币+肥尾"(round12/13 已证),本轮要么找到**条件性**edge,要么诚实确认无。
- 强平方向数据落 holdout,本轮只能探索性观察、不可验证。
- 衍生品特征覆盖期短(约 03-26 起),相关结论样本更少、更弱。
- 2 个月单一行情 regime,任何 holdout 通过的发现仍属"暂定",上实盘前需 forward paper-shadow。
